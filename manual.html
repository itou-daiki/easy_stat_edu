初学者の皆さまへ
「分析ソフトを入れたけれど、データが読み込めなくて分析すら始まらない」「用語が難しくてどのボタンを押せばいいかわからない」——easyStatは、そんな「つまづき」をなくすために作られました。
このマニュアルは、専門用語を極力減らし、「やりたいこと」から直感的に機能を探せるようにしています。

【最重要】正しいデータの作り方

分析ソフトが一番苦手なのは「人間が見るためだけに綺麗に装飾されたExcel」です。
コンピューターが読み込めるように、以下のルールを守ってデータ（Excel・CSV）を作りましょう。

  • 1行目は「データの項目名（変数名）」だけを書く
  • セルの結合（くっつけること）は絶対にしない！
  • 1人のデータは「横に1行」で書く
  • 数字の列に「約」や「kg」などの文字を混ぜない！

ダメな例（エラーになります）

アンケート結果 身体測定
ID 性別 体重
1 男 約60kg
2 女

× 1行目と2行目がタイトルになっている
×「アンケート結果」などがセル結合されている
× 体重に「kg」という文字が混ざっている

正しい例（こちらを目指す）

ID 性別 体重
1 男 60
2 女 55

〇 1行目がシンプルな項目名になっている
〇 単位は含めず数字だけを書いている
〇 1人につき綺麗な1行になっている

「数値データ」と「カテゴリデータ」の違い

easyStatでは、手法を選ぶと「数値変数が〇つ必要」「カテゴリカル変数が〇つ必要」と表示されます。
この2つの違いを勘違いすると、「データを入れたのに分析ボタンが押せない！」というトラブルになります。

量的データ（数値）

「足し算や平均・大小に意味がある数字」です。

  • 身長 (160cm, 175cm)
  • テストの点数 (80点, 95点)
  • アンケートの5件法 (1:全く違う ～ 5:そう思う)

質的データ（カテゴリ）

「グループ分け・名前・記号」のことです。見た目が数字でも、平均に意味がなければカテゴリです。

  • 性別 (男, 女)
  • 所属クラス (A組, B組)
  • 要注意: 「1=男, 2=女」と入力した数字（平均の1.5に意味がないため）

統計用語の基礎：p値と有意差

この先、色々な表で p値 (p-value) という数字が出てきます。
これは「たまたま偶然、この結果になった確率」を表します。難しい数式は知らなくて大丈夫です。

これだけ覚えてください！

p値 が 0.05 より小さい (p < 0.05) かどうかを探します。
0.05より小さければ、「偶然起きたにしては珍しすぎる！よって意味のある違いだ！」と判断します（これを有意差があると言います）。見つけやすいように * のマークが付きます。

AI分析サポーター

目的：「どう分析すればいいか見当もつかない」時に、AI（システム）が自動でできそうな分析を提案してくれます。

使い方：データを読み込んだら、まずはこれを開いてみてください。「この分析を使う」ボタンでそのまま画面へ移動できます。

データの整形・欠損処理

目的：分析を始める前に、データをきれいにしたり、必要な列だけを抽出したりします。
（「空白があるとめちゃくちゃになる分析」のために、事前に空白行を消したりします）

データの結合

目的：「事前テストのファイル」と「事後テストのファイル」の2つを、出席番号などを目印にして1つのファイルにくっつけます。

因子得点の算出

目的：「Q1, Q2, Q3...」というアンケートの点数を足し算して、「総合満足度」のような新しい列（得点）を作ります。

探索的データ分析・グラフ (EDA)

目的：点数の平均が何点か、ばらつきはどのくらいかなど、まずは手持ちのデータの全体像と形を見たい。

数値 1つ以上

結果の見方

ボタンを押すだけで、ヒストグラム（山の形）、箱ひげ図、平均値などの表が一気に出力されます。異常な値が入っていないかのチェックにも使います。

2つのグループを比べる (t検定 / U検定)

目的：「男性と女性」「A組とB組」など、2つのグループで、点数（平均点）に意味のある差があるか調べたい。

どれを使えばいい？

  • 対応なし t検定： 全くの別グループ（男女など）を比べる王道パターン。点数の列 と 男女の列 を選びます。
  • 対応あり t検定： 同じ人たちの「事前テストと事後テスト」を比べるパターン。事前点数 と 事後点数 を選びます。
  • マン・ホイットニーのU検定（ノンパラメトリック）： データが極端に偏っていて、平均値で比べるのが怪しい時のt検定の代わり。順位（どちらが上か）で戦わせます。

3つ以上のグループを比べる (分散分析 / K-W検定)

目的：「A組、B組、C組」など、3つ以上のグループで点数に差があるか調べたい。

どれを使えばいい？

  • 一要因分散分析 (ANOVA)： 3つ以上のクラス間で点数を比べる王道パターン。点数の列 と クラスの列 を選びます。
  • 二要因分散分析： 「勉強法の違い(A/B)」×「性別(男女)」の2つの原因が、どう影響し合っているか（交互作用）を検証します。
  • クラスカル・ウォリス検定（ノンパラメトリック）： データに偏りがある時のANOVAの代用品です。

結果の見方： まず分散分析表の p値 で「どこかの組に差がある」ことを見つけます。その後、下に出てくる「事後検定（多重比較）」の表を見て、「具体的にA組とC組の間に差がある」という結論を出します。

アンケートの偏りをみる (クロス集計 / カイ二乗)

目的：「性別（男女）」によって「きのこ/たけのこ派」の割合に違いがあるか？を調べたい。

カテゴリ名 2つ以上

どれを使えばいい？

  • クロス集計： まずは「男性の犬好きは何人？」のように人数の内訳（表と帯グラフ）を出したい時。
  • カイ二乗検定： その人数の偏りが「偶然なのか、本当に性別と好みが関連している（意味のある偏り）のか」を p値 で白黒つけたい時。
  • フィッシャーの正確確率検定： カイ二乗検定と同じですが、人数が数人〜数十人と極端に少ない時に使います。
  • マクネマー検定： 同じ人たちの「賛成・反対」が、イベントの前後でどう変化したかを調べる時に使います。

関係性を調べる (相関分析)

目的：「身長が高い人ほど、体重も重いか？」のように、右上がりの関係か、右下がりの関係かを調べたい。

数値 2つ以上

見方： 相関係数 (r) という数字（-1から1の間）を見ます。1に近いほど強い正の相関（連動して増える）、-1に近いほど負の相関（一方が増えれば減る）です。

未来を予測する・影響度を知る (回帰分析)

目的：「勉強時間」や「過去の成績」をもとに、未来の「テストの点数」を推測する数式を作りたい。

どれを使えばいい？

  • 単回帰分析： 原因(X)が1つ、結果の数値(Y)が1つの場合。（例: 勉強時間 → テスト点数）
  • 重回帰分析： 原因(X)が複数ある場合。結果の表の「標準化偏回帰係数(β)」を見ると、どの原因が一番テストの点数に強い影響を与えたかのランキングが作れます。
  • ロジスティック回帰推分析： 結果(Y)が「合格/不合格」のような2択（カテゴリ）の場合。その確率を予測します。

たくさんの情報をスッキリまとめる (因子 / PCA)

目的：たくさんの質問項目を、「文系力」「理系力」といった見えないテーマでまとめたり、総合的な新しい指標を作りたい。

どれを使えばいい？

  • 因子分析： 「国・数・英…」の点数から背後に潜む力（国語と英語ができる人は「言語能力因子」が高い傾向）を発見します。主にアンケート分析で使います。
  • 主成分分析 (PCA)： 全ての点数をギュッと圧縮して、「総合評価点」という1つの軸を作りたい時などに使います。

時系列 / テキストマイニング

  • 時系列データ分析： 「1年間の毎月の売上」など、時間経過のトレンドや周期を見ます。数値が必要です。
  • テキストマイニング： アンケートの「自由記述欄」に書かれた文章を解析し、よく使われている単語を大きな文字（ワードクラウド）で表示します。テキストが入った列を選びます。

よくあるトラブル・FAQ

困ったときは、ここをチェック！
ファイルをアップロードしても読み込まれない（赤いエラーが出る）
一番上の行以外にタイトルが書いてあったり、セルが結合されている可能性があります。
いったんExcelを開き、1行目に項目名、2行目以降にデータという最もシンプルな形になっているか確認してください。
分析画面で、「分析を実行」ボタンが暗くて押せない！
その分析に必要な「変数（列）」がすべて選択されていません。
たとえば「t検定」なら、数値を1つ、カテゴリを1つ（または数値を2つ）リストから選んで青く反転させる必要があります。
テストの点数（数値）を変数として選びたいのに、リストに出てこない
その列のどこかに文字（「約」や「kg」など）が混ざっていたり、空欄がありすぎるため、システムが「カテゴリデータ」または「文字」だと勘違いしています。
Excelに戻って、該当する列がすべて半角の数字だけになっているか確認してください。
パラメトリックとかノンパラメトリックって何？
パラメトリック＝「データの形が普通の山型（ベルカーブ）で、平均値で比べても大丈夫」な時の一般的な手法。
ノンパラメトリック＝「超すごい人が1人だけいて平均値がおかしくなっている」など、データが歪んでいる時に順位を使って比べる安全な手法です。
